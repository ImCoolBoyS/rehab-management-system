# AI Agent 开发工程师技能图谱 & 精神康复系统 AI Agent 设计方案

## 一、AI Agent 开发工程师所需核心技能（2025-2026）

> 基于 2025-2026 年行业需求调研，AI Agent 开发是增长最快的细分方向之一，
> 薪资溢价约 30-50%，属于全栈 + AI 的交叉能力岗位。

### 1. 编程与基础架构层

| 技能 | 说明 | 推荐工具/框架 |
|------|------|-------------|
| Python | AI Agent 主力语言，数据/模型/API 全栈贯通 | FastAPI, Pydantic, asyncio |
| TypeScript | 前端 Agent UI + Node.js Agent 运行时 | React, Next.js, Vercel AI SDK |
| 数据库 | 结构化数据 + 向量检索 | PostgreSQL + pgvector, Redis, ChromaDB |
| 容器化 | 环境一致性/部署 | Docker, Docker Compose, K8s |
| API 设计 | RESTful + Streaming + WebSocket | OpenAPI, SSE, WS |

### 2. LLM 应用开发核心

| 技能 | 说明 | 推荐工具 |
|------|------|---------|
| 提示工程 | Chain-of-Thought, Few-shot, System Prompt | LangChain, 自研模板 |
| Function Calling | LLM 调用外部工具的桥梁 | OpenAI/Claude/DeepSeek API |
| RAG | 检索增强生成 - 让 LLM 基于私有数据回答 | LlamaIndex, LangChain, pgvector |
| 多模态 | 图片/PDF/音频理解 | GPT-4o, Gemini, Qwen-VL |
| 模型微调 | 领域知识注入 | LoRA, QLoRA, Unsloth |

### 3. Agent 框架与架构设计

| 技能 | 说明 | 代表框架 |
|------|------|---------|
| 单 Agent | 工具调用 + 推理循环 | LangChain, OpenAI Assistants API |
| 多 Agent 协作 | 分工/辩论/验证 | LangGraph, CrewAI, AutoGen |
| ReAct 模式 | 推理-行动-观察循环 | 所有 Agent 框架的基础模式 |
| 记忆系统 | 短期/长期/实体记忆 | Mem0, Zep, 自建向量DB |
| MCP 协议 | Model Context Protocol - 工具标准化 | Anthropic MCP SDK |

### 4. 工程化与可观测性

| 技能 | 说明 |
|------|------|
| LLM 可观测性 | LangSmith, LangFuse - 追踪 Token/延迟/质量 |
| 评估体系 | 构建测试集，自动化评估回复质量 |
| 安全防护 | Prompt Injection 防护、数据脱敏、RBAC 权限 |
| 流式输出 | SSE/WebSocket 实现打字机效果 |

### 5. 领域知识转化能力

- 将业务规则转化为 Agent 可理解的 **Tool Definition**（函数描述）
- 将领域文档构建为 **Knowledge Base**（RAG 知识库）
- 设计 **Workflow**（工作流：数据采集 → 分析 → 报告 → 归档）

### Agent 功能矩阵

| 功能模块 | 输入 | 处理方式 | 输出 | 优先级 |
|---------|------|---------|------|-------|
| **数据概况** | 无（自动） | 统计SQL汇总 | 卡片式仪表盘: 学员数/评估数/训练数/探访数 | P0 |
| **智能问答** | 自然语言问题 | 意图识别 → SQL查询 → 格式化 | 结构化答案 + 可视化 | P0 |
| **风险预警** | 无（自动扫描） | 多条件规则匹配 | 高风险学员名单 + 预警原因 | P0 |
| **评估分析** | 学员ID或范围 | 分数分布/趋势分析 | 分析报告 + 建议 | P1 |
| **训练推荐** | 学员ID | 基于评估分数的规则匹配 | 个性化训练建议 | P1 |
| **探访优化** | 探访记录 | 频率/效果分析 | 探访计划优化建议 | P1 |
| **报告生成** | 时间段/站点 | 多维度数据汇总+模板 | Markdown/PDF报告 | P2 |
| **PDF 智能解析** | 评估PDF | OCR+LLM提取结构化数据 | 自动填入评估分数 | P2 |
| **多Agent协作** | 复杂任务 | 规划→分工→执行→汇总 | 综合报告 | P3 |

### 分阶段实施路线图

#### 第一阶段：Rule-based AI 助手（2-3天）— 你现在可以立即开始

**目标**：无需任何 LLM API，纯规则引擎驱动

**新增文件**：
- server.py - 新增 POST /api/v1/ai/chat
- src/components/AIAssistant.tsx - 新增 AI 助手 UI 组件
- src/lib/api.ts - 新增 aiChat API 函数
- src/App.tsx - 注册新 Tab

**规则引擎核心逻辑**（意图识别 + SQL 生成）：
```
用户输入: "本月新增了多少学员？"
  → 关键词匹配: [学员, 新增, 本月]
  → 意图: count_students_by_month
  → SQL: SELECT COUNT(*) FROM students WHERE created_at >= date_trunc('month', CURRENT_DATE)
  → 输出: "本月新增 X 名学员"
```

**规则覆盖 20+ 常见问题**，涵盖：
- 学员统计（总数/新增/按站点/按风险等级）
- 评估分析（分数分布/完成率/趋势）
- 训练统计（类型分布/完成情况）
- 探访统计（频率/精神状态分布）
- 综合查询（跨表关联）

#### 第二阶段：LLM 增强（1-2天）

**目标**：接入 LLM API，支持自然语言理解

**新增依赖**：
```
pip install openai langchain
```

**增强内容**：
1. 意图识别从规则匹配升级为 LLM 分类
2. 复杂问题直接调用 LLM 生成答案（附加上下文数据）
3. 支持对话记忆（多轮上下文）

**API Key 配置**（.env 文件）：
```
LLM_API_KEY=sk-xxx
LLM_MODEL=gpt-4o-mini
LLM_BASE_URL=https://api.openai.com/v1
```

#### 第三阶段：高级 Agent 能力（3-5天）

**目标**：多 Agent 协作、RAG、自动化工作流

1. **Multi-Agent 架构**
   - 数据分析师 Agent（SQL 查询 + 统计）
   - 报告撰写 Agent（自然语言生成）
   - 审核 Agent（质量控制）

2. **RAG 知识库**
   - 将康复政策文档、评估标准 PDF 导入向量库
   - 支持基于政策文件的问答

3. **自动化工作流**
   - 定时扫描风险学员 → 自动生成预警报告
   - 评估结果 → 自动生成康复建议
   - 探访记录 → 智能安排下次探访时间

### 技术选型理由

| 选择 | 理由 |
|------|------|
| **FastAPI SSE** | Python 原生异步支持，无需额外消息队列 |
| **关键词+分类器混合** | 第一阶段零成本，第二阶段无缝升级 LLM |
| **复用 API 而非直连 DB** | 权限/RLS 一致性，审计日志自动生效 |
| **React Server-Sent Events** | 打字机效果体验好，实现简单 |
| **pgvector** | 与现有 PostgreSQL 复用，无需新基础设施 |
